# 產品需求文件 (PRD)
**專案:** AI 虛擬教師培訓平台 (AI Virtual Teacher Training Platform)
**狀態:** 草稿 (Draft)
**版本:** 1.1
**日期:** 2026-02-28

## 1. 執行摘要 (Executive Summary)

### 1.1 願景與目標 (Vision & Goal)
本文件定義了「AI 虛擬教師培訓平台」的產品需求。系統的核心目標是建立一個低風險、可重複的沙盒環境，讓教育工作者能透過與 AI 虛擬學生互動，練習**薩提爾溝通模式 (Satir Communication Model)** 與**社會情緒學習 (SEL)**。平台將提供專家級、由 LLM 驅動的量化與質化回饋，以縮短教師技能養成的週期。

### 1.2 非目標 (Non-Goals / Out of Scope)
為確保 MVP 階段的交付品質與專注度，以下項目**不在**本次發佈範圍內：
*   支援多人線上同時進入同一個教室（僅限單機一對一模擬）。
*   支援自訂上傳 PDF 擴充 RAG 知識庫（MVP 採用系統內建的固定知識庫）。
*   第三方 LMS (Learning Management System) 或 SSO (如 Google Workspace, Microsoft AD) 的深度整合。

### 1.3 成功指標 (Success Metrics / KPIs)
*   **使用者參與度:** 測試階段每位教師平均每週完成 >= 3 次完整培訓會話。
*   **系統效能:** 語音對話端到端延遲 (End-to-End Latency) P90 <= 3 秒。
*   **回饋實用性:** 教師對 AI 回饋報告的滿意度評分 (CSAT) 達 4.0/5.0 以上。

## 2. 使用者畫像 (User Personas)
*   **主要目標：教師 (Sarah)**
    *   **背景:** 擁有 2-10 年經驗的 K-12 教育工作者。
    *   **痛點:** 面對學生情緒失控時缺乏即時應對經驗；傳統同儕觀課回饋成本高且不夠客觀。
    *   **期望:** 隨時隨地能進行短時間 (5-10分鐘) 的情境演練，並獲得具體的話術改進建議。

## 3. 核心功能需求 (Functional Requirements - MVP P0)

| ID | 功能模組 | 使用者故事 (User Story) | 驗收標準 (Acceptance Criteria) |
|---|---|---|---|
| **FR-1** | 身份與安全 | 作為教師，我希望能安全登入系統，保護我的練習隱私。 | 1. 支援 Email/Password 註冊與登入。<br>2. 採用 JWT 進行 API 授權。<br>3. 密碼採用 bcrypt 等強雜湊演算法儲存。 |
| **FR-2** | 會話生命週期 | 作為教師，我能自由控制模擬流程（開始/暫停/結束）。 | 1. 提供明確的狀態切換按鈕。<br>2. 會話意外中斷 (如網路斷線) 能在 5 分鐘內恢復狀態。 |
| **FR-3** | 雙向語音即時互動 | 作為教師，我能用說話的方式與系統溝通，並聽到學生的回應。 | 1. 前端支援 VAD (語音活動偵測) 或 Push-to-Talk。<br>2. 系統能即時轉錄教師語音並生成學生語音回應。 |
| **FR-4** | 動態虛擬人設 | 作為教師，我希望面對不同個性的學生，且能看見其情緒變化。 | 1. 內建至少 3 種性格 (如：挑戰型、退縮型)。<br>2. 前端 2D 紙娃娃能根據 LLM 輸出的情緒標籤即時切換表情。 |
| **FR-5** | 專家評估儀表板 | 作為教師，我能在結束後收到符合 SEL/Satir 理論的評分。 | 1. 產出包含雷達圖 (多維度指標) 的報告。<br>2. 提供關鍵逐字稿標註，並給予「原本怎麼說更好」的具體建議。 |
| **FR-6** | 重練與歷史追蹤 | 作為教師，我能快速重試特定情境，並檢視過去的進步軌跡。 | 1. 報告頁提供「One-click 再練一次」。<br>2. 歷史列表支援依日期與分數排序。 |

## 4. 非功能性需求 (Non-Functional Requirements)

| 類別 | 需求標準 |
|---|---|
| **效能 (Performance)** | - STT + LLM + TTS 的總體處理時間 P90 需 < 3 秒。<br>- 非 AI 依賴的 API 端點 P99 回應時間 < 300ms。 |
| **可用性 (Availability)** | - 系統整體 SLA 目標為 99.9%。<br>- 前端需具備優雅降級 (Graceful Degradation) UI，在生成延遲時顯示「思考中」。 |
| **安全性 (Security)** | - 資料傳輸強制使用 TLS 1.2+。<br>- 遵循最小權限原則，防範 OWASP Top 10 漏洞（特別是 Prompt Injection 防護）。 |
| **合規與隱私 (Privacy)** | - 音訊檔案在轉錄完成後，快取保留不超過 24 小時。<br>- 逐字稿資料需進行去識別化處理 (PII Masking) 方可用於後續模型微調。 |
